## ADR 60초 말하기 카드

> ⏱️ **60초** 말로 설명

### 1. 우리는 무엇을 선택했나?

- **백엔드**: Node.js + Express로 서버를 구성하고, Claude API(`@anthropic-ai/sdk`)를 연동했다.
- **응답 방식**: 일반 REST 응답 대신 **SSE(Server-Sent Events)** 스트리밍 방식을 선택했다.
- **AI 모델**: `claude-opus-4-7`을 사용하고, 시스템 프롬프트에 **prompt caching**(`cache_control: ephemeral`)을 적용했다.

---

### 2. 다른 선택지는 무엇이었나?

| 항목 | 선택한 것 | 대안 |
|------|-----------|------|
| 응답 방식 | SSE 스트리밍 | 단순 REST (전체 응답 후 반환) |
| 모델 | claude-opus-4-7 | claude-sonnet, claude-haiku |
| 인증/세션 | 없음 (익명) | JWT 로그인, 세션 관리 |
| 데이터 저장 | 인메모리 Map (24h TTL) | DB (MongoDB, SQLite 등) |

---

### 3. 우리 팀의 기준에서 왜 이 선택이 더 낫나?

- **SSE 스트리밍**: 자기소개서 첨삭 결과는 텍스트가 길기 때문에, 전체 응답을 기다리면 UX가 나쁘다. 스트리밍으로 글자가 실시간으로 출력되면 사용자가 기다리는 느낌을 덜 받는다.
- **Prompt caching**: 동일한 시스템 프롬프트를 매 요청마다 전송하는 비용을 줄이기 위해 캐싱을 적용했다. API 비용과 응답 속도 모두 개선된다.
- **인메모리 저장**: 공유 링크 기능은 24시간만 유효한 임시 기능이므로, DB 도입 없이 Map으로 충분히 구현 가능하다. 복잡도를 낮추는 것이 팀 기준에 맞다.

---
